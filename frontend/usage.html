Why SSL Notify Me?

With new automated services like the ones offered by Let's Encrypt or AWS Certificate Manager, the pain of manually managing certificates (including deployment and renewal) is greatly diminished. Nevertheless, there are many webmasters out there who still have to deal with certificates authorities and expiring SSL certificates. SSL Notify Me is meant for those admins, and in general for anyone who doesn't want to risk forgetting to renew a certificate.

How it works

  1. You create a monitoring check clicking on the Remind Me button in the home page
  2. You confirm your address is valid clicking on the link you received via email
  3. Every morning at 6:00 UTC, the system will run a check for you against sslexpired.info APIs, and if the check fails, an alert will be sent to your email address with a summary of the problem
  4. Once the check has been activated, it will run every day, meaning that if you don't issue a new certificate or fix the problem affecting the https server hosting the SSL certificate, you will keep receiving the same alert every morning, hence the alert comes with an unsubscribe link which you can follow to remove the check if it's too noisy. Don't worry if you remove the check by mistake, you can register any time you want following this procedure

Limitations

  • At the moment, it's possible to have only one domain checked per email account, so if you will register a new check with the same address, it will overwrite any existing one
  • Non-ascii domains (i.e. with non-English chars like факитол.рф) are not currently supported
